# گزارش ممیزی فنی و امنیتی dicodePing

تاریخ بازبینی: **۲۰۲۶-۰۷-۱۲**
نسخه: **0.1.2**

## دامنه

کد منبع کلاینت Windows و Android، منطق دریافت subscription، سنجش اتصال، راه‌اندازی Xray، مسیرهای VPN/TUN، رفتار رابط کاربری، زنجیرهٔ تأمین و workflowهای CI/Release بررسی شدند. باینری‌های ناشناختهٔ آرشیو اولیه اجرا نشدند و از مخزن عمومی حذف شدند.

## نتیجه

- ۲۷ تست واحد و نگه‌داری موفق است.
- quality gate بدون خطا و هشدار عبور می‌کند.
- ۵۲ فایل XML/Manifest اندروید و برابری ۱۲۵ منبع فارسی/انگلیسی اعتبارسنجی شده است.
- سنجش نهایی سرور از مسیر واقعی SOCKS/Xray انجام می‌شود؛ TCP فقط پیش‌آزمون محدود است.
- Android هر دو مسیر پیش‌فرض IPv4 و IPv6 را داخل VPN می‌برد و خطر bypass مستقیم IPv6 بسته شده است.
- آیکون Windows در resource فایل EXE، سطح Qt، `AppUserModelID` و HWND کوچک/بزرگ اعمال می‌شود.
- coreهای بومی داخل سورس commit نمی‌شوند؛ workflow نسخه‌های ثابت را دریافت و قبل از ساخت SHA-256 را کنترل می‌کند.
- Release شامل checksum، SBOM استاندارد SPDX، متن مجوزهای ثالث و artifact attestation است.

## محدودیت قابل‌بیان

ممیزی ایستا و تست واحد جای آزمون واقعی Windows 10/11 و دستگاه‌های Android را نمی‌گیرد. ساخت نهایی فایل‌ها عمداً در GitHub Actions انجام می‌شود و ماتریس پذیرش دستگاه واقعی در [`TEST_MATRIX.md`](TEST_MATRIX.md) ثبت شده است.
