# dicodePing v0.1.5-rc.1

این نسخه یک پیش‌انتشار برای Windows، Linux و Android است و روی پایداری اتصال، انتخاب بهینه‌تر سرور، توقف امن اتصال و تجربه کاربری یکدست‌تر تمرکز دارد.

## تغییرات تازه در v0.1.5-rc.1

### اصلاح فوری دریافت سرور در Windows و Linux

- mirrorهای منبع پیش‌فرض به‌جای انتظار زنجیره‌ای، هم‌زمان آزمایش می‌شوند تا خرابی DNS یا کندی یک دامنه کل دریافت را متوقف نکند.
- مسیر مستقل GitHub API اضافه شد و پاسخ Base64 آن داخل برنامه به subscription واقعی تبدیل می‌شود.
- یک snapshot سالم از منبع پیش‌فرض داخل بسته‌های Windows و Linux قرار می‌گیرد؛ اگر همه مسیرهای شبکه و cache شکست بخورند، فهرست اولیه همچنان قابل دریافت است.
- اولین پاسخ معتبر فوراً استفاده می‌شود و عملیات کند باقی‌مانده، رابط کاربری و دریافت سرورها را معطل نمی‌کند.

### پایداری اتصال در Windows و Linux

- برای هر تلاش اتصال یک پرچم لغو مستقل ساخته می‌شود؛ فرمان «قطع اتصال» دیگر با شروع هم‌زمان یک اتصال تازه بازنویسی یا بی‌اثر نمی‌شود.
- شروع، توقف، پاک‌سازی پردازش Xray، مسیرهای مستقیم، فایل‌های موقت و رابط TUN با قفل چرخه‌عمر هماهنگ شده‌اند.
- انتخاب خودکار دیگر علاقه‌مندی کاربر را به‌عنوان امتیاز سرعت حساب نمی‌کند؛ سرور واقعاً سریع‌تر در اولویت قرار می‌گیرد.
- شکست‌های اخیر همچنان امتیاز منفی محدود دارند تا یک سرور ناپایدار پشت سر هم انتخاب نشود.
- گزینه‌های جایگزین اتصال خودکار از IPهای متفاوت انتخاب می‌شوند؛ چند نام یا پورت از یک شبکه، صف تلاش مجدد را اشغال نمی‌کند.
- سرورهای زیر ۷۰ میلی‌ثانیه همچنان فقط برای اتصال دستی قابل استفاده‌اند و وارد اتصال خودکار نمی‌شوند.
- سرورهای تشخیص‌داده‌شده در ایران در جدول کم‌رنگ و غیرفعال هستند، دکمه اتصال ندارند و وارد صف خودکار نمی‌شوند.

### پایداری و انتخاب خودکار در Android

- قانون حداقل ۷۰ میلی‌ثانیه به انتخاب خودکار Android نیز اضافه شد و منطق هر سه پلتفرم یکسان شد.
- سرورهای دارای موقعیت ایران در Android هم برای انتخاب، اتصال دستی و اتصال خودکار غیرفعال شدند.
- حالت خودکار Android اکنون تا پنج سرور سالم و واقعاً آزمایش‌شده را به‌ترتیب پینگ امتحان می‌کند؛ شکست یک سرور پایان کل عملیات نیست.
- صف تلاش مجدد از شبکه‌ها و IPهای متفاوت ساخته می‌شود تا چند کانفیگ تکراری از یک سرور پشت سر هم امتحان نشوند.
- توقف هسته بومی VPN از Thread اصلی سرویس خارج شد تا لمس «قطع اتصال» باعث مکث رابط یا ANR نشود.
- پاک‌سازی هسته، TUN و شبکه زیرین هم‌زمان‌سازی شد تا شروع، توقف و نابودی Service روی یکدیگر اجرا نشوند.
- سرویس موقعیت سوم به تشخیص IP در Android اضافه شد تا در صورت محدودیت یا خطای دو سرویس قبلی، پوشش بیشتری وجود داشته باشد.

### رابط کاربری و تجربه استفاده

- ردیف سرورهای ایران در Windows، Linux و Android به‌صورت واضح غیرفعال نمایش داده می‌شود.
- دکمه‌های اتصال سرور غیرفعال، متن و پیام توضیحی روشن دارند.
- انتخاب سرور غیرفعال از مسیرهای اتصال صفحه خانه، فهرست و دکمه اتصال انتخابی مسدود شده است.
- تغییرات زنده پینگ همچنان بدون حذف فهرست یا نمایش دوباره Skeleton انجام می‌شود.
- وضعیت تلاش اتصال خودکار با تغییر واقعی وضعیت اتصال هماهنگ است و پس از موفقیت یا قطع دستی، صف باقی‌مانده پاک می‌شود.

### به‌روزرسانی و انتشار

- شماره داخلی Android به `20` و نسخه نمایشی هر سه پلتفرم به `0.1.5` ارتقا یافت.
- بررسی نسخه جدید پیش از بررسی تغییرات ساب انجام می‌شود تا اعلان برنامه پشت تأخیر منبع‌ها نماند.
- فایل درست هر پلتفرم مستقیماً از Release باز می‌شود.
- نسخه جدید مستقیماً با برچسب صحیح `v0.1.5-rc.1` منتشر می‌شود و هیچ شماره ساختگی بالاتری کنار آن ساخته نمی‌شود.
- هر سه خروجی همراه با `SHA256SUMS` توسط GitHub Actions ساخته و منتشر می‌شوند.

# تاریخچه کامل تغییرات از v0.1.2 تا v0.1.5

## v0.1.2 — اولین انتشار عمومی

- انتشار عمومی Windows و Android با رابط فارسی و انگلیسی، پوسته روشن و تیره و مدیریت subscriptionها.
- اتصال TUN در Windows و VPN مبتنی بر `VpnService` در Android.
- پشتیبانی Android از IPv4 و IPv6 و جلوگیری از نشت IPv6 خارج از VPN.
- تست TCP و بررسی واقعی HTTPS از مسیر Xray پیش از معتبر دانستن اتصال.
- انتخاب خودکار بر اساس پاسخ، پینگ، موقعیت و سلامت واقعی مسیر.
- جلوگیری از انتخاب خودکار پینگ‌های کمتر از ۷۰ میلی‌ثانیه و سرورهای فاقد موقعیت معتبر در Windows.
- رد اتصال Windows در صورتی که TUN اجرا شده ولی ترافیک واقعی عبور نمی‌کند.
- نمایش پینگ اتصال، دانلود، آپلود، دامنه‌های مستقیم و برنامه‌های خارج از VPN.
- دریافت هسته Xray و Wintun با نسخه و SHA-256 ثابت.

## نکات نصب

- Windows برای ساخت و پاک‌سازی TUN به دسترسی Administrator نیاز دارد.
- Linux برای TUN باید از اسکریپت همراه بسته و دسترسی لازم استفاده کند.
- APK با گواهی پایدار پروژه امضا می‌شود. اگر Android خطای تداخل بسته نشان داد، نسخه قبلی نصب‌شده با امضای متفاوت را حذف و سپس APK جدید را نصب کنید؛ حذف برنامه داده‌های محلی آن نسخه را نیز پاک می‌کند.
- موقعیت سرور برآورد شبکه‌ای IP است و مکان فیزیکی دقیق را تضمین نمی‌کند.
