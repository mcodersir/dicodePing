# dicodePing v0.1.3-rc.5

Pre-release focused on desktop resilience and Linux availability.

- Windows and Linux now reject stale or mismatched Xray cores before TUN startup.
- Core downloads are atomic, verified with streaming SHA-256, and never leave a partial archive as valid state.
- Auto mode retries up to five ranked servers when a reachable endpoint fails the real routed-traffic check.
- Unexpected Xray termination is reflected in the UI and cleans up the owned tunnel state.
- Server discovery, refresh, selected-server ping and list deletion are locked while connected to prevent misleading tunneled probes and state corruption.
- Linux x86_64 ships as a portable GUI bundle using the same desktop code and pinned Xray TUN core as Windows.
- Android application logic is unchanged; only `versionCode` advances to 8.
