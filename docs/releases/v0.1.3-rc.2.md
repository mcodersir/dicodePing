# dicodePing v0.1.3-rc.2 — پیش‌انتشار

این نسخه روی اصلاح تجربه Windows و دقیق‌ترشدن شناسایی سرورها تمرکز دارد.

## اصلاحات Windows

- نام اصلی کانفیگ از `ps` در VMess و Fragment در VLESS/Trojan/Shadowsocks خوانده می‌شود؛ نام‌های «نامشخص» و نام‌های ساختگی قدیمی دیگر حفظ نمی‌شوند.
- سنجش سرورها از ترکیب ICMP و اتصال واقعی TCP به پورت کانفیگ استفاده می‌کند؛ سرورهایی که ICMP را مسدود کرده‌اند همچنان قابل شناسایی هستند.
- برای دامنه‌های چند IP، چند آدرس بررسی می‌شود و IP قابل دسترس‌تر انتخاب می‌شود.
- ظرفیت نگهداری سرورها افزایش یافته تا کانفیگ‌های بیشتری از منابع پرحجم بررسی و نمایش داده شوند.
- تشخیص موقعیت با چند IP و چند سرویس GeoIP دوباره تلاش می‌شود؛ در نبود داده قطعی، پرچم یا نام کشور داخل نام کانفیگ فقط به‌عنوان راهنمای نمایشی استفاده می‌شود و وارد انتخاب خودکار بهترین سرور نمی‌شود.
- پینگ زیر 70 میلی‌ثانیه همچنان برای انتخاب خودکار معتبر نیست، اما تست TCP باعث می‌شود سرورهای واقعی بیشتری شناسایی شوند.
- پینگ سرور متصل از طریق مسیر فعال TUN/Xray و درخواست واقعی اینترنت اندازه‌گیری می‌شود، نه اتصال مستقیم به IP سرور.
- جدول «بهترین سرورها» در صفحه خانه قابل انتخاب است؛ با کلیک سرور انتخاب و با دوبار کلیک اتصال آغاز می‌شود.
- هنگام قطع اتصال، رابط بلافاصله وضعیت «در حال قطع اتصال» نشان می‌دهد و پاک‌سازی Xray/TUN در Thread جدا انجام می‌شود؛ برنامه دیگر بدون واکنش به نظر نمی‌رسد.

## فایل‌های نصب

# dicodePing v0.1.3-rc.2 — Pre-release

This release candidate focuses on Windows server discovery, latency accuracy, and connection UX.

## Windows fixes

- Preserves real configuration names from VMess `ps` and URL fragments.
- Combines ICMP with real TCP endpoint probing, allowing servers that block ICMP to remain detectable.
- Tries multiple resolved IPv4 addresses and retries geolocation through the existing multi-provider resolver.
- Measures connected latency through the active TUN/Xray route using a real network request.
- Makes the Home page best-server table selectable and double-click connectable.
- Shows immediate disconnecting feedback while Xray/TUN cleanup runs outside the UI thread.
- Keeps implausible sub-70 ms results out of automatic best-server selection.
