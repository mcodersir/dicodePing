# dicodePing v0.1.2 — اولین انتشار عمومی

این نسخه، اولین انتشار عمومی dicodePing برای Windows و Android است؛ یک کلاینت سریع، متن‌باز و دو‌زبانه برای مدیریت و اتصال به کانفیگ‌های مبتنی بر Xray.

## بهبود مهم اتصال خودکار

- در Android، هنگام اولین اجرا و بروزرسانی دوره‌ای دو روزه، فقط فهرست سرورها دریافت نمی‌شود؛ تعدادی از گزینه‌های مناسب پیش از خروج از Splash با مسیر واقعی Xray آزمایش و مکان‌یابی می‌شوند.
- اتصال خودکار Android دیگر به سرور تست‌نشده یا فاقد لوکیشن برنمی‌گردد.
- در Windows، پینگ‌های غیرواقعی کمتر از ۷۰ میلی‌ثانیه و سرورهای فاقد لوکیشن معتبر وارد انتخاب «بهترین سرور» نمی‌شوند.
- Windows فقط زمانی وضعیت Connected را نمایش می‌دهد که علاوه بر پاسخ اینترنت، افزایش واقعی ترافیک در شمارنده‌های TUN/Xray مشاهده شود.
- حالتی که برنامه در Windows 11 متصل نشان داده می‌شد اما هیچ سایتی باز نمی‌شد، دیگر به‌عنوان اتصال موفق پذیرفته نمی‌شود.

## قابلیت‌های اصلی

- دریافت، مدیریت و بروزرسانی subscriptionها و کانفیگ‌های پشتیبانی‌شده
- بررسی اولیه TCP و تست واقعی HTTPS از داخل مسیر Xray
- انتخاب خودکار سرور با توجه به latency، jitter، لوکیشن و سلامت واقعی اتصال
- اتصال TUN در Windows و VPN مبتنی بر `VpnService` در Android
- پشتیبانی از IPv4 و IPv6 در مسیر VPN اندروید
- نمایش وضعیت اتصال، پینگ و میزان مصرف دانلود و آپلود
- امکان تعریف دامنه‌ها و برنامه‌های خارج از تونل
- رابط فارسی و انگلیسی با حالت روشن و تاریک
- دریافت و بررسی خودکار Xray، Wintun و هسته Android با نسخه و SHA-256 ثابت

## دانلود

- **Android:** `dicodePing-v0.1.2-android.apk`

برای اطمینان از اصالت فایل‌ها، مقدار آن‌ها را با `SHA256SUMS` مقایسه کنید.

## نکات نصب

### Windows

برنامه برای ایجاد رابط TUN به دسترسی Administrator نیاز دارد. ممکن است Windows SmartScreen در اولین اجرا پیام هشدار نمایش دهد؛ نام فایل و SHA-256 را با همین Release بررسی کنید.

### Android

فایل APK برای Android 7 و نسخه‌های جدیدتر ساخته شده است. در اولین اتصال، مجوز VPN سیستم‌عامل باید توسط کاربر تایید شود.

# dicodePing v0.1.2 — First Public Release

dicodePing is an open-source bilingual Xray client for Windows and Android, focused on reliable server validation, responsive connection handling, and a clean user experience.

## Auto-connect reliability hotfix

- Android now performs real Xray-path latency checks and geolocation during the first launch and the two-day subscription refresh before automatic connection is offered.
- Android automatic mode no longer falls back to an untested or unlocated server.
- Windows excludes implausible sub-70 ms host pings and servers without a verified location from best-server selection.
- Windows reports a successful connection only after real traffic movement is observed through the TUN/Xray counters.
- A tunnel that starts but cannot pass user traffic is now rejected instead of being displayed as connected.
