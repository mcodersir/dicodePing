# dicodePing v1.7.0-rc.2

## خلاصه

اصلاح پینگ‌گیری ICMP، بهبود تشخیص کیفیت، پیاده‌سازی واقعی VPN برنامه‌محور در اندروید، و پیاده‌سازی واقعی اشتراک VPN.

## پینگ‌گیری ICMP

ماژول `icmp_ping.py` اضافه شد. پینگ‌گیری اکنون از دستور سیستمی `ping` استفاده می‌کند که بسته‌های واقعی ICMP Echo Request ارسال می‌کند. این روش در ویندوز و لینوکس بدون نیاز به دسترسی root کار می‌کند. در صورت عدم دسترسی به دستور `ping`، از raw socket به‌عنوان fallback استفاده می‌شود.

تابع `net.py::icmp_ping` نیز به‌روزرسانی شد تا ابتدا از دستور سیستمی `ping` استفاده کند و سپس از پیاده‌سازی قبلی (Windows IcmpSendEcho API یا raw socket لینوکس) به‌عنوان fallback استفاده کند.

## تشخیص کیفیت

الگوریتم `rate_quality` به‌روزرسانی شد. اکنون علاوه بر پینگ، از jitter (انحراف معیار نمونه‌های ICMP) و تعداد خطاها برای تعیین کیفیت استفاده می‌کند:

- عالی: پینگ ≤ ۱۵۰ms، jitter ≤ ۲۰ms، ۰ خطا
- خوب: پینگ ≤ ۳۵۰ms، jitter ≤ ۵۰ms، ≤ ۲ خطا
- متوسط: پینگ ≤ ۸۰۰ms، jitter ≤ ۱۰۰ms، ≤ ۵ خطا
- ضعیف: پینگ > ۸۰۰ms یا بدون پینگ یا خطای زیاد

## VPN برنامه‌محور (اندروید)

پیاده‌سازی واقعی در `DicodeVpnService.kt` با استفاده از `addAllowedApplication` و `addDisallowedApplication`:

- حالت غیرفعال: همه برنامه‌ها از VPN استفاده می‌کنند (پیش‌فرض)
- حالت allowlist: فقط برنامه‌های انتخابی از VPN استفاده می‌کنند
- حالت denylist: برنامه‌های انتخابی از VPN استفاده نمی‌کنند

تنظیمات در `SettingsStore.kt` ذخیره می‌شوند و از `MainActivity.kt` به VPN service ارسال می‌شوند.

## اشتراک VPN

ماژول `vpn_sharing.py` اضافه شد. پیاده‌سازی واقعی برای ویندوز و لینوکس:

- ویندوز: استفاده از `netsh routing ip nat` برای فعال‌سازی NAT روی رابط TUN
- لینوکس: استفاده از `iptables` برای MASQUERADE و FORWARD، و فعال‌سازی IP forwarding

تنظیمات اشتراک USB و هات‌اسپات در `SettingsStore.kt` (اندروید) و `settings.json` (دسکتاپ) ذخیره می‌شوند.

## پلتفرم‌ها

تمام تغییرات در ویندوز، لینوکس و اندروید پیاده‌سازی شد.
